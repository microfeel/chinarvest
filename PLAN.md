## 0. 外贸认知——架构的设计原点

传统 B2B 农产品出口网站 = 产品目录 + 联系方式 → 等客户上门

**我们的认知不同：**

| 维度 | 传统做法 | 我们的做法 |
|------|---------|-----------|
| **市场选择** | 凭经验/关系决策 | **海关数据驱动**——实际进口数据告诉你哪国买什么、谁在买、多少钱 |
| **获客方式** | 等买家搜索找过来 | **GEO + AI Agent 友好**——让 ChatGPT/Claude/Gemini 搜索时主动推荐我们的产品 |
| **客户处理** | 所有询盘同等对待 | **背景调查分层**——确认是真客户再做投入（样品、报价、跟进） |

核心洞察：**信息不对称就是利润来源。海关数据解决"卖去哪"，GEO 解决"怎么被找到"，客户甄别解决"谁值得服务"。三者闭环就能让一个微型团队撬动多国市场。**

## 2. 三大核心能力详解

### 2.1 海关数据驱动的市场选择（战略层）

**问题：** 我们有几十种农产品，每个产品适合哪些市场？不可能每个市场都铺。

**方案：** 用海关提单数据回答这些问题：

| 问题 | 数据要什么 | 决策输出 |
|------|----------|---------|
| 五常大米哪些国家进口量最大？ | HS 1006（大米）+ 目的国 + 数量 | 决定大米主推：🇺🇸？🇲🇾？🇸🇬？ |
| 这些国家的中国大米进口是增长还是下降？ | 同比进口量 | 增长中 → 加大投放；下降 → 分析原因 |
| 谁在进口？（具体买家名） | 提单上的 consignee | 精准名单用于内容定向 |
| 他们现在从哪个供应商进货？ | 提单上的 shipper | 找到替代机会，针对性优化优势对比 |
| 进口价格区间？ | 单价（金额÷数量） | 我们的定价是否有竞争力 |

**数据源全景对比：**

| 平台 | 类型 | 覆盖国家 | 数据更新 | API 是否开放 | 价格区间（估） | 优点 | 缺点 |
|------|------|---------|---------|------------|-------------|------|------|
| **腾道 (Tendata)** | 国内代理 | 200+ | 月度 | 封闭/高价 | ¥30K-100K/年 | 中文界面+支持，国内客户多 | 数据源二次加工，延迟高，价格贵 |
| **特易 (Teyi)** | 国内代理 | 180+ | 月度 | 封闭/高价 | ¥30K-80K/年 | 同上 | 同上，数据落后于源头 |
| **驭岳** | 国内代理 | 200+ | 月度 | 封闭 | ¥20K-60K/年 | 相对便宜 | 数据质量和完整性存疑 |
| **tradedata.pro (CIC)** | 境外直采 | 80+（提单级） | 周度/月度 | ✅ REST API | 约 $500-2000/月（估） | 新加坡公司，API 开放，数据结构完整 | 价格中高 |
| **Volza** | 境外直采 | 70+ | 周度 | ✅ API | $100-500/月 | 性价比最高，70+国家，API 开放 | 部分国家数据深度不够 |
| **ImportGenius** | 境外直采 | 21+（深度） | 实时 | ✅ API | $200-500/月 | 美国数据最强，实时更新 | 覆盖国家少 |
| **Descartes Datamyne** | 境外直采 | 60+ | 周度 | ✅ 企业API | $500-2000/月 | 亚太数据强 | 贵 |
| **Panjiva (S&P)** | 境外直采 | 全球 | 实时 | ✅ 企业API | $15K+/年 | 数据最全 | 极贵，不适合初创 |

**核心判断（与你之前文档一致）：** 国内平台从境外源头采购再转售，代价是数据滞后+价格高+无 API+字段二次加工有损。**境外直采是正确路径。**

**推荐策略：**

```
第1步：注册 Volza 免费试用（性价比最高，$100-500/月）
       ├─ 查询核心 HS 编码在各国的进口数据
       ├─ 输出第一份《品类-市场优先级报告》
       └─ 同时注册 tradedata.pro 试用，对比数据质量

第2步：确定 Top 3 目标市场
       ├─ 决定独立站语言优先级
       └─ 识别目标市场中的活跃买家名单

第3步：如 API 需求扩大，评估 tradedata.pro 或 Descartes Datamyne
```

**为什么首选 Volza：**
- 价格最低（$100-500/月），初创阶段负担得起
- 70+ 国家覆盖，对农产品够用
- API 开放，可与客户甄别系统集成
- 可以先用试用版验证数据质量，满意再付费

**输出：** `/data/intelligence/market-priority-report.md`

### 2.2 GEO 深度优化（战术层）

**我们的定位：** 不仅要被 Google 搜到，更要被 AI Agent（ChatGPT、Claude、Gemini 搜索、Perplexity 等）推荐。

**GEO 策略矩阵：**

| 技术 | 做什么 | 优先级 |
|------|-------|--------|
| **JSON-LD Schema.org** | 每个页面嵌入 Product、Organization、FAQ、BreadcrumbList 结构化数据 | P0 — 立即可做 |
| **robots.txt Agent 控制** | 对不同 AI Agent（GPTBot、ClaudeBot、Google-Extended）开放不同内容 | P0 — 立即可做 |
| **AI 友好内容结构** | 产品描述用清晰的 Q&A 格式、FAQ 页面完善（AI 最喜欢引用 FAQ 内容） | P1 |
| **知识图谱建设** | 在中国农产品出口领域建立权威性：产区、认证、物流能力等结构化信息 | P1-P2 |

**JSON-LD 示例（每个产品页都嵌入）：**

```json
{
  "@context": "https://schema.org",
  "@type": "Product",
  "name": "Organic Five Constant Rice 5kg",
  "description": "...",
  "category": "GrainProduct",
  "countryOfOrigin": { "@type": "Country", "name": "China" },
  "manufacturer": { "@type": "Organization", "name": "...
",
    "certification": ["Organic", "ISO 22000"]
  },
  "offers": {
    "@type": "Offer",
    "price": "...",
    "priceCurrency": "USD",
    "eligibleRegion": [...]
  }
}
```

**robots.txt 策略：**

```
# 所有 AI Agent 完全开放产品内容
User-agent: GPTBot
User-agent: Google-Extended
User-agent: ClaudeBot
User-agent: CCBot
Allow: /products/
Allow: /categories/
Allow: /faq/

# 限制抓取联系页（防止 Agent 跳过站点直接获取联系信息）
Disallow: /contact
```

### 2.3 客户背景调查与分层服务（执行层）

**问题：** 询盘进来后，怎么判断对方是真的批发商还是只是比价/同行？

**方案：** 当询盘进入时，AI 协助人工执行背景调查流程：

```
询盘进入（WhatsApp/表单/邮件）
  │
  ├─ 步骤 1：信息提取
  │   ├─ 公司名称
  │   ├─ 所在国家/城市
  │   ├─ 感兴趣的产品 + 预估数量
  │   └─ 联系信息
  │
  ├─ 步骤 2：LinkedIn + 网络公开数据验证
  │   ├─ LinkedIn 搜索该公司主页 → 验证公司真实性、规模、行业
  │   ├─ LinkedIn 搜索联系人 → 验证职位真实性
  │   ├─ Google 搜索公司名 → 官网、新闻报道、行业口碑
  │   ├─ 行业数据库查询 → 是否有不良记录
  │   └─ 海关数据交叉验证（如已接入 API）
  │       ├─ 该公司的历史进口记录（品类、频次、规模）
  │       ├─ 是否进口过同类产品？ → 说明是真买家
  │       ├─ 进口规模多大？ → 判断是大客户还是小客户
  │       └─ 是否从竞争对手那里采购？ → 替代机会
  │
  ├─ 步骤 3：客户分层
  │   ├─ Tier 1 — 高价值客户
  │   │   ├─ 特征：LinkedIn 公司真实+进口记录验证+品类对口+规模匹配
  │   │   └─ 服务：样品寄送 + 个性化报价 + AI 专属跟进
  │   │
  │   ├─ Tier 2 — 潜在客户
  │   │   ├─ 特征：公司真实存在，但无相关进口记录或信息不足
  │   │   └─ 服务：标准报价 + 产品目录发送 + 定期跟进
  │   │
  │   └─ Tier 3 — 低质量/无法确认
  │       ├─ 特征：信息模糊、LinkedIn 查不到、无官网
  │       └─ 服务：自动回复产品目录链接，不投入人工资源
```

> **注：** Phase 1 阶段客户背调以手工 + LinkedIn 为主。Phase 3 接入海关数据 API 后可实现自动化验证。

**样品寄送流程（仅 Tier 1 触发）：**

```
客户确认 → 生成样品清单

  ├─ 我方内部：
  │   ├─ 从供应商调取样品（或我们备货）
  │   ├─ 准备产品规格书 + 认证文件复印件
  │   ├─ 安排国际快递（DHL/FedEx 到付或我方预付）
  │   └─ 提供追踪号给客户

  └─ 客户跟进：
      ├─ 样品到达确认
      ├─ 客户反馈收集
      └─ 如满意 → 推进首单
```

## 4. 分阶段实施路线图

### Phase 1 — 基础能力（4周）

| **D — 样品流程** | 样品寄送管理 |
